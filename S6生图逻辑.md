# S6 封面·配图 生成逻辑

> 适用页面：`流水线.html`（生成）、`成稿预览.html` / `合规自检.html`（同套卡渲染）、样式在 `assets/atelier.css`。
> 最近更新：2026-06

---

## 一、S6 只产出两类「卡」

| 类型 | 是什么 | 由谁生成 |
|---|---|---|
| **长图笔记卡（主推/默认）** | 苹果备忘录风的 **CSS 文字卡**（非图片，零乱码、最稳） | `planNotePage()` 排版 |
| **AI 图片卡** | 封面 / 氛围底图（真·图片） | `regenImageCustom()` / `genImageFromPlan()` 调 Seedream / 可灵 |

---

## 二、默认路由（跑 S6 / 一键全套时）

入口：`generateCoverImagesCore(n)`，在 S6（cover 步）完成后由 `autoRun()` 自动调用，张数对齐对标（`coverCount()`）。

每张对标判断：

```js
const textHeavy = isTextHeavyRef(对标这张);
return { i, textHeavy, isNote: textHeavy };   // 默认：文字密集 → 笔记卡
```

- **文字密集的对标**（清单 / 干货 / 表格 …）→ **长图笔记卡**（绝大多数走这条）
- **对标本身是实拍照片** → AI 图片卡
- 设计原则：干货内容默认出**备忘录长图笔记卡**，不强行 AI 画图（AI 画不出精确文字排版）

---

## 三、长图笔记卡（`planNotePage`）逻辑

- **内容来源**：唯一来自 **S3 框架「配图规划」第 N 张**（`frameSliceForImage(N)`），不取 S4 正文。
- **容错**：AI 失败自动重试 1 次；两次都失败 → 兜底（清洗掉「作用 / 生图提示词」等元信息，只留文字要点，仍套备忘录风）。
- **标题**：封面标题直接采用 S3 配图规划第 1 张的文字原文（不自创、不改写，超 14 字才精简）。
- **拆页**：单页过重自动拆成多页，**续页不重复主标题**（无「（续）」）。

### 备忘录风（`is-memo`）

触发：智能体「配图视觉风格」或「配图风格库模板」里出现 *备忘录 / 便签 / 手机截图 / memo* → 整篇套**仿真 iPhone 截图**：

- 顶部状态栏：`9:41` + 信号 / WiFi / 电量（纯 SVG）
- 黄色「‹ 备忘录」导航栏，**日期居中**显示（`memoDateStr()`）
- 底部工具栏（表格 / 清单 / Aa / 相机 / 标记笔）+ Home 指示条
- SF / 苹方字体、黄色荧光高亮
- 外壳固定不缩放，正文区在中间自动适配（`fitNote`）

### 背景色（前端直接解析颜色词，不靠 AI）

在「配图视觉风格」里写颜色词即生效（`planNotePage` 内 `memoBg`）：

| 关键词 | 色值 |
|---|---|
| 米白 / 奶油 / 米色 | `#fffdf5` |
| 青 / 浅青 / 青绿 / cyan / teal | `#e2f4f3` |
| 黄 / 浅黄 / 便签黄 / 鹅黄 | `#fff8d6` |
| 灰 / 浅灰 | `#f2f2f4` |
| 蓝 / 浅蓝 | `#eaf1fb` |
| 粉 / 浅粉 | `#fdeef0` |
| 绿 / 薄荷 | `#ebf6ee` |
| 紫 / 浅紫 | `#f1ecfb` |
| 橙 / 杏色 / 米黄 | `#fdeede` |
| 直接写 `#RRGGBB` | 精确取该色 |

优先级规则：

- **配图（内容页）= 统一**用 `memoBg`（所有内容页同色）
- **封面 = 可自定义**：优先跟 S3 规划色（`s.bg`），否则用 `memoBg`
- 不写颜色 = **白底**（`#ffffff` 默认）
- 顶栏 / 底栏始终白色（像真 iOS UI），只有纸面变色

## 四、AI 图片卡（封面 / 氛围底图）逻辑

**入口**：无独立「生成配图」按钮。每张卡下方按钮按类型区分：

- 笔记卡 → 「🔄 重排这页」
- 图片卡 → 「🔄 重新出图」

点开统一弹框 `openRegenModal(i)`，三种操作：

1. **重排文字**（笔记卡）：填补充要求，重排这页文字
2. **AI 出图**：写画面提示词 → `regenImageCustom()` 出图替换
3. **上传真实图**：直接用原图（不盖文字 / 不蒙层、不经 AI，秒出）

**出图供应商**：封面 → Seedream(ark)，内页 → 可灵(kling)。
**风格注入**：`buildImgPrompt()` 把「配图视觉风格」文字放最前，统一画风。

### 垫图 / 图生图（i2i）

- 没上传参考图 → **文生图**（风格用文字注入）
- 上传参考图 + 勾选「直接用原图」→ 直接用，精确秒出
- 上传参考图 + 取消勾选（AI 参考重画）→ 走 i2i
  - ⚠️ 当前 ark 账号 **SeedEdit 未开通**，i2i 会失败 → **服务端自动降级为文生图**（`server.js` `/api/image` ark 分支：i2i 失败时丢掉 `image` 重试 t2i），保证能出图，但不精确复刻参考图
  - 要真正 i2i：在火山方舟开通 SeedEdit，配 `SEEDEDIT_MODEL`

---

## 五、配图风格库的作用

- 「配图风格库」每张样图可「✨提炼」成一句画风文字
- **笔记卡**：风格库模板文字 = 文字排版风格（配色 / 版式）最高优先级（`pickStyleRef`）
- **AI 图片卡**：风格库 / 配图视觉风格 = 画风文字注入提示词

## 六、关键函数索引（流水线.html）

| 函数 | 作用 |
|---|---|
| `generateCoverImagesCore(n)` | S6 总生成入口，路由 note / image |
| `planNotePage(i, refVis, isCover, title, t, extra)` | 排一页长图笔记卡（含 memo / memoBg） |
| `noteCardHTML(page)` | 渲染笔记卡 HTML（含 memo 外壳） |
| `noteStyleVars(page)` | 输出卡片 CSS 变量（bg/accent/hl/dark） |
| `splitHeavyPage` / `splitPageInTwo` | 过重页拆分（续页不重复标题） |
| `openRegenModal(i)` | 统一「重排 / 重出图 / 上传」弹框 |
| `regenImageCustom(i, prompt, refDataUrl)` | 按提示词(+可选垫图)出图 |
| `pickStyleRef(plan)` | 从风格库选最优模板 |
| `isTextHeavyRef(refVis)` | 判定对标是否文字密集（决定 note/image） |

样式：`assets/atelier.css` 中 `.cv-note` / `.cv-note.is-memo` 等。
