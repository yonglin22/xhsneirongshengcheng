# 能量向量中间层 · 数据契约（v1.0，选品引擎的地基）

> 机器可用文件：`../契约/energy_vector.ts`（类型/枚举/合成/校验）、`../契约/energy_vector.schema.json`（运行时校验）。
> 本文是人可读的契约说明：为什么这么设计、各 agent 怎么写、怎么合成、选品引擎怎么读。**照它建，四个智能体才能合成「一次推荐」。**

---

## 〇、为什么要这一层

没有它：五行命理、塔罗、星盘/合盘各出一段话，你有**四个各玩各的玄学玩具**。
有了它：三路解读的出口统一成**一种中间语言**（五行·能量向量 + 议题 + 关系轴），选品引擎只干一件事——把向量映射到打了同样标签的 SKU。于是你有**一串「为你而排」的珠**。

**单一可信源**：五行只有一个来源（八字），议题只有一张受控词表，谁都不能越界、不能自由造词。

```
[输入] 出生信息 / 塔罗抽牌 / 合盘 / 处境
   │
[解读 agent 各写各的段] ── 只写授权字段（WRITE_SCOPE）
   ├─ 五行命理/八字 → her.wuxing (+ ta.wuxing)   ← 五行唯一真值
   ├─ 塔罗           → her.themes / ta.themes
   ├─ 星盘(本命)     → her.themes / ta.themes
   ├─ 合盘           → relation + her/ta.themes + scene
   └─ 情绪(可选)     → her.themes
   │
[合成] mergeThemes：同议题聚合、去重、忌神优先
   │
[能量向量 EnergyVector]  ← 单一可信源
   │
[选品引擎 selectBeads] 向量 → SKU 序列（+ 推荐理由）
```

---

## 一、结构总览（字段语义）

以 `EnergyVector`（见 `energy_vector.ts`）为准，关键字段：

| 字段 | 含义 | 谁写 |
|---|---|---|
| `her.wuxing` | 五行盈缺：scores(0-100) + lack/excess + **favorable(喜用)** / **avoid(忌神)** | 五行命理 |
| `her.themes[]` | 议题标签：`{theme, weight0-1, source}` | 塔罗/星盘/合盘/情绪 |
| `her.taboo / prefer` | 禁忌（色/材质）、偏好（色/预算） | 输入或 agent |
| `ta` | TA 的同结构向量 | 合盘时 |
| `relation` | 关系轴：summary(文案) + needs(her/ta 各补的议题) + palette_hint + **anchorTheme(关系锚)** | 合盘 |
| `scene` | ambiguous/together/reconcile/single | 合盘/输入 |
| `meta` | generatedAt / agents / requestId | 系统 |

## 二、受控词表（不得自由造词）

**五行 Element**：`jin 金 · mu 木 · shui 水 · huo 火 · tu 土`

**议题 Theme（14 项，agent 的 themes 只能从这里取）**

| 议题 | 含义（触发场景） |
|---|---|
| 情感决断 | 关系里犹疑、要拿主意（暧昧去留、复合与否） |
| 情绪安定 | 情绪起伏大、内耗、易被放大 |
| 沟通表达 | 表达/接收错频、说不出口 |
| 信任松绑 | 猜忌、控制、不敢信 |
| 自我价值 | 自我怀疑、讨好、边界模糊里的自我 |
| 桃花招缘 | 招正缘/桃花 |
| 关系承诺 | 承诺、稳定、长期 |
| 边界建立 | 边界、拒绝、课题分离 |
| 疗愈放下 | 放下过去、疗愈旧伤 |
| 专注定力 | 分心、定力、执行 |
| 睡眠安神 | 睡眠、安神 |
| 焦虑纾解 | 焦虑、紧绷、纾解 |
| 事业推进 | 事业方向/推进（非 P0 主线，留位） |
| 财务稳健 | 与钱的关系/稳健（红线：不碰投资标的） |

> 新增议题 = 升 minor 版本 + 同步补「议题→配石/香方」映射表，否则选品引擎读到会落空。

## 三、各 agent 写入契约（越界即校验失败）

**通则**：只写 `WRITE_SCOPE` 授权的字段；themes 只取受控词表；不写不属于自己的段。

- **五行命理 / 八字（wuxing）**：写 `her.wuxing`（合盘再写 `ta.wuxing`）。给 scores + lack/excess + **favorable/avoid**。**禁**写 themes/relation。是五行唯一真值来源。P0 可用简化盘 `source:'bazi_simplified'`，接口不变，P1 换精确。
- **塔罗（tarot）**：把解读落成 `themes`（如 现状宝剑三逆→`情感决断`）。**禁**写 wuxing。
- **星盘·本命（natal）**：写 `themes`（如 月亮双鱼→`情绪安定`）。**禁**写 wuxing。
- **合盘（synastry）**：写 `relation`（summary + needs + palette_hint + anchorTheme）+ 双方 `themes` + `scene`。**禁**写 wuxing。
- **情绪陪伴（emotion，可选）**：写 `her.themes`（`焦虑纾解/睡眠安神/情绪安定`）。

*示例（合盘 agent 片段）*
```json
{ "relation": { "summary": "她要被回应，他要学会表达", "needs": { "her": ["情感决断"], "ta": ["沟通表达"] }, "palette_hint": "warm", "anchorTheme": "情绪安定" },
  "her": { "themes": [ { "theme": "情感决断", "weight": 0.8, "source": "synastry" } ] } }
```

## 四、合成规则

1. **themes 聚合**（`mergeThemes`）：同一议题多来源 → `weight = 1 − ∏(1 − w_i)`（软饱和，越多来源越强但不超 1），按 weight 降序。
2. **her + ta**：各自独立成串；`relation.needs` 把「关系里要补的议题」并入对应人的 themes（weight 兜底 0.6）。
3. **忌神优先（红线）**：选品补五行只在 `favorable`/`lack` 里选，**绝不落在 `avoid`（忌神）**——即使某石对应的议题很匹配，若其五行属忌神则排除。
4. **缺 wuxing 降级**：未接八字 → `wuxing` 缺省，选品退化为**纯 themes 选石**，结果标 `degraded:true`（前端可提示「补出生信息可按五行精选主石」）。

## 五、选品映射契约（向量 → SKU）

选品引擎 `selectBeads(person, {wristCm, budget, anchorTheme})`（签名见 ts）。规则（与 PRD §四一致）：

**① 主石 · 补五行**（按 favorable/lack 首位）

| 需补 | 主石候选 | 色系 |
|---|---|---|
| 火 huo | 南红玛瑙 / 石榴石 / 红纹石 | 红 |
| 木 mu | 绿幽灵 / 东陵玉 / 翠榴石 | 绿 |
| 水 shui | 海蓝宝 / 蓝纹玛瑙 / 月光石 | 蓝/白 |
| 金 jin | 白水晶 / 银发晶 / 黑曜石 | 白/黑 |
| 土 tu | 黄水晶 / 茶晶 / 虎眼石 | 黄/棕 |

**② 配石 · 议题**

| 议题 | 配石 |
|---|---|
| 情绪安定 | 月光石 / 粉晶 |
| 沟通表达 | 蓝纹玛瑙 / 海蓝宝 |
| 情感决断 | 南红 / 石榴石 |
| 信任松绑 | 黑曜石 / 黑碧玺 |
| 桃花招缘 | 粉晶 / 草莓晶 |
| 焦虑纾解/睡眠安神 | 香承香珠（见③） |

**③ 香珠 · 香方功用**（表述为**气味体验**，非疗效）：焦虑/内耗→「解郁凝神」香方；睡眠→「安神」香方；提振→「醒神」香方。

**④ 关系锚**：两串各嵌一颗 `relation.anchorTheme` 对应的**同款珠**（默认静心香珠），象征同频。

**⑤ 结构/颗数**：主石 1–3 + 配石 + 隔珠 + 香珠 1，按 `wristCm`（净手围）定总颗数与串长。

**⑥ 过滤**：去 `taboo`（色/材质）、控 `prefer.budget`。

**⑦ 可解释**：每串产出一句 `reason`（模板：`{relation.summary 或议题}——这串补的是{element_cn}：{主石}{功用}，{香珠}{气味功用}。`）。

## 六、校验与版本

- 运行时按 `energy_vector.schema.json` 校验；业务红线用 `validateEnergyVector`（忌神/喜用冲突、议题词表、weight 范围、有 ta 必有 relation）。
- **版本**：`version` 固定校验；加字段/加议题=minor（向后兼容 + 补映射）；改语义/删字段=major（配迁移）。选品引擎按 version 分支。

## 七、完整示例

**单人（single，未接八字→降级）**
```json
{ "version": "1.0", "scene": "single",
  "her": { "themes": [ { "theme": "情绪安定", "weight": 0.7, "source": "tarot" },
                        { "theme": "专注定力", "weight": 0.4, "source": "natal" } ] },
  "meta": { "generatedAt": "2026-07-04T12:00:00Z", "agents": ["tarot","natal"], "requestId": "req_001" } }
```

**情侣（couple，含八字+合盘，对齐蓝图 29 岁问感情范例）**
```json
{ "version": "1.0", "scene": "ambiguous",
  "her": {
    "wuxing": { "scores": { "jin": 40, "mu": 55, "shui": 82, "huo": 22, "tu": 35 },
      "lack": ["huo","tu"], "excess": ["shui"], "favorable": ["huo","tu"], "avoid": ["shui"],
      "dayMaster": "mu", "source": "bazi_simplified" },
    "themes": [ { "theme": "情感决断", "weight": 0.86, "source": "synastry" },
                { "theme": "情绪安定", "weight": 0.6, "source": "tarot" } ] },
  "ta": {
    "wuxing": { "scores": { "jin": 60, "mu": 30, "shui": 45, "huo": 50, "tu": 58 },
      "lack": ["mu"], "excess": [], "favorable": ["mu"], "avoid": ["jin"], "source": "bazi_simplified" },
    "themes": [ { "theme": "沟通表达", "weight": 0.7, "source": "synastry" } ] },
  "relation": { "summary": "她要被回应，他要学会表达",
    "needs": { "her": ["情感决断"], "ta": ["沟通表达"] },
    "palette_hint": "warm", "anchorTheme": "情绪安定" },
  "meta": { "generatedAt": "2026-07-04T12:00:00Z", "agents": ["wuxing","tarot","synastry"], "requestId": "req_002" } }
```

*选品引擎读上例 → 她的串：南红(补火)+黄水晶(补土)+月光石(情绪安定)+静心香珠(锚)+隔珠；TA 的串：绿幽灵(补木)+蓝纹玛瑙(沟通表达)+静心香珠(锚)。忌神红线：她 avoid 水→不选海蓝/蓝纹当她的主石；TA avoid 金→不选白水晶/银发晶。*
