# P0 漏斗 · PRD — 合盘 → 关系对串（可直接交付开发）

> 配套：《关系合盘·优化落地包》（合盘怎么算/怎么解读/出什么向量）+ `synastry_reader.html`（能跑的 3D 合盘参考实现）+ `bracelet_studio`（照片合成成串台）。
> **这份 PRD 只回答一件事：把「合盘 → 关系对串」这条能赚钱的漏斗从入口到复购跑通。** 它是 P0 的单一可信源。
> P0 的成败裁决数就一个：**LTV / CAC > 1**（每获客一个人的贡献能不能盖过获客成本）。其余都是它的输入。

---

## 一、一句话目标 & P0 边界

**目标**：用一条漏斗验证「命理推荐能不能转成真金白银的下单」。
**漏斗**：合盘钩子 → 输入两个人 → 出合盘解读 + 契合分 → **为你俩排的对串** → 成串预览/微调 → 加购 → 支付 → 私域沉淀 → 复购。

**P0 只做（In Scope）**
- 关系合盘：输入 → 真盘计算 → 跨盘相位 → 解读(含能量向量) → 契合分。
- 选品引擎（规则版）：向量 → SKU 序列 → 生成「你的串 / TA 的串」。
- 成串预览（照片合成）+ 净手围/包装/证书选项 + 加购 + 微信支付。
- 积分/钱包 + 扣点时序 + 订单 + 支付回调 + 定制品退换规则。
- 私域钩子 + 复购召回。
- 埋点 + P0 裁决看板。

**P0 不做（留接口，P1/P2）**
单人本命串 · 会员订阅 · 多轮追问 · 完整自由 DIY · 1 对 1 人工解读（仅留导流钩子）· 八字五行 agent 的精确盘（P0 用简化版，见 §四）· 海外 ToC。

## 二、核心漏斗（逐屏 + 转化目标）

```
[入口钩子] 首页/星盘页「关系合盘·测测你和TA的契合度」为第一 CTA
    │  ← 埋点 funnel_enter
[S1 输入] 她 + TA 出生信息(年月日时地) · TA可「不知时辰」降级 · 场景三选一 · 一句话处境
    │  ← 校验积分余额；不足→充值。埋点 synastry_start
[S2 计算·仪式] 双盘计算 + 跨盘相位（loading 用双轮叠合动画盖住等待，做成仪式感）
    │
[S3 合盘结果] 契合分环 + 关系定性 + 各自带来什么 + 合拍/消耗 + 提示 + 小行动
    │  ← 核心「被说中」体验。埋点 synastry_done
    │  末尾醒目卡：为你俩排了一串 →
[S4 对串推荐] 你的串 / TA 的串（3D/照片合成预览）+ 推荐理由(取自解读) + 价格
    │  ← 埋点 reco_view（关键转化观测点）
[S5 成串预览/微调] 收拢成串 + 可加珠/换珠(微调，非空白DIY) + 净手围 + 包装/证书
    │  ← 埋点 preview_enter
[S6 加购] 加入购物车（记录串配方+选项+价格）
    │  ← 埋点 add_cart
[S7 结算支付] 确认订单 → 微信支付 → 支付成功
    │  ← 埋点 pay_success（变现达成）
[S8 沉淀/复购] 引导加企微/公众号解锁「关系养护提醒」；订单完成后 N 天召回下一次合盘
    └  ← 埋点 join_private / repurchase
```

**每步转化率都埋点**，重点盯三个坎：**S3→S4→S6→S7**（解读→看推荐→加购→支付）。哪一步掉得狠，就改哪一步，别整体推倒。

## 三、页面地图（逐屏规格）

### S1 · 合盘输入
- **她**：出生日期、时间、城市（内置经纬度+时区）。**TA**：同上 + 「不知道出生时辰」勾选（勾选→降级：只用日月金火、不给宫位/上升，页面明示「侧重性格契合，补时辰解锁完整版」= 二次转化钩子）。
- **场景**：暧昧 / 在一起 / 分手复合（决定分场景分寸，见落地包）。
- **处境**：一句话（可选，≥0 字）。
- **CTA**：开始合盘。触发前校验积分，不足→充值页。
- 免责固定展示：「仅供自我觉察与关系参考，不构成对关系走向的确定性预测。出生信息仅用于排盘，可删除。」

### S2 · 计算仪式
- 前端展示双轮叠合/星空动画（招牌动作），后端并行：算两盘 → 算跨盘相位 → 调 AI 出解读 JSON → 算契合分 → 跑选品引擎。
- 全程一次 loading 盖住；**扣点在此阶段按 §五时序处理**。

### S3 · 合盘结果（核心体验）
- 契合分环（55–92，见落地包评分口径）+ 一句关系定性。
- 结构化解读：各自带来什么(她/TA) · 合拍点 · 消耗点 · 提示 · 小行动（渲染落地包 System Prompt 输出的 JSON）。
- 可交互 3D 星盘（点行星出详解）。
- 末尾**推荐卡**：「按你俩的盘，我给你们各排了一串 →」跳 S4。
- 导私域/1对1 钩子（林老师·姻缘复合，高客单人工层，仅导流）。

### S4 · 对串推荐
- Tab：你的串 / TA 的串。每串：成串预览图 + 主石构成 + **推荐理由文案**（取自解读的 dynamic + relation_axis，见 §四映射）+ 价格。
- 「同款关系锚珠」高亮说明（两串共通那颗）。
- CTA：预览/微调 → S5；或直接加购 → S6。

### S5 · 成串预览 / 微调
- 收拢成串（照片合成，`bracelet_studio` 那套）。
- **微调而非空白 DIY**：可加珠/换珠（从推荐衍生的候选池，非全库自由拖），系统保证仍符合命理逻辑（换主石会提示「偏离你补火的建议」）。
- 净手围（决定颗数/串长）+ 包装（普通/礼盒）+ 证书（购买/不购买）。
- 实时更新价格。

### S6–S7 · 加购 / 结算支付
- 购物车项 = {串配方(SKU序列) + 净手围 + 包装 + 证书 + 价格快照 + 关联 synastryId}。
- 结算 → 确认（地址、串长、建议净手围）→ 微信支付 → 成功页。
- 成功页：预计工期（现货/定制分开标，香承定制约 15–20 天）+ 加私域解锁养护提醒。

### S8 · 私域 / 复购
- 支付成功 & 解读末尾双入口导企微/公众号。
- 复购召回：订单完成后 N 天推「关系近况变了？再合一次盘」；连续关怀触达。

## 四、选品引擎（向量 → SKU）—— P0 规则版

> 这是「选品系统」的心脏。P0 用**规则映射**（可解释、可维护、够验证），P1 再谈个性化/学习。
> 入参 = 落地包定义的合成向量：`{her:{wuxing,themes}, ta:{wuxing,themes}, relation_axis}`。
> **P0 五行来源**：先用出生信息做**简化八字**（日主+四柱五行计数，判最缺/最忌），未接完整八字 agent 也能出；接口预留，P1 替换为精确盘。

**SKU 标签模型**（每个石头/香珠预先打标）：
`{id, 名称, 类型(主石/配石/隔珠/香珠/吊坠), 五行, 色系, 议题功用[], 价位, 现货/定制}`

**映射规则（按顺序执行）**

| 步骤 | 规则 | 示例 |
|---|---|---|
| ① 主石·补五行 | 取该人最缺/最需补的 element → 对应主石 | 火→南红/石榴石/红纹石；木→绿幽灵/东陵；水→海蓝宝/蓝纹/月光；金→白水晶/银发晶；土→黄水晶/茶晶/虎眼 |
| ② 配石·议题 | 每个 theme → 功用配石 | 情绪安定→月光石/粉晶；沟通表达→蓝纹玛瑙/海蓝宝；情感决断→南红/石榴石；信任松绑→黑曜石/黑碧玺；招桃花→粉晶/草莓晶 |
| ③ 香珠·香方功用 | 由 theme 映射香承香方（**表述为气味体验，非疗效**） | 焦虑/内耗→「解郁凝神」香方；睡眠→「安神」香方；提振→「醒神」香方 |
| ④ 关系锚 | 两串各嵌一颗**同款**（同香珠或同色主石）= 同频象征 | 你的串 & TA 的串各一颗静心香珠 |
| ⑤ 结构/颗数 | 主石1–3 + 配石 + 隔珠 + 香珠1，按**净手围**定总颗数与串长 | 净手围14cm→约14颗 |
| ⑥ 过滤 | 去禁忌（如忌某色/材质）、控预算区间 | — |

**输出** = 两串 SKU 序列 + 每串一句推荐理由（模板：`{关系定性}——这串给你补的是{缺的element对应词}：{主石}{功用}，{香珠}{气味功用}。`）。

> 落地包已说明：**五行来自八字、议题来自星盘/塔罗、关系轴来自合盘**，三路各出一段，选品引擎在这里汇总。

## 五、数据模型 & 计费闭环

**实体（字段可增）**
- `birthprofile`：userId, role(self/ta), 出生UTC, 经纬度, 时区, 知否时辰。
- `chart`：profileId, 行星黄经/星座/宫位(JSON), 上升, 算法版本。
- `synastry`：herChartId, taChartId, 相位矩阵(JSON), score, 解读(JSON), energy向量(JSON), sceneType, ts。
- `braceletplan`：synastryId, side(her/ta), skuSeq[], 净手围, 包装, 证书, 价格快照, reason。
- `sku`：见 §四标签模型；含现货/定制、拿货价/售价。
- `wallet` / `ledger`（充值/消费/赠送/返还四类）/ `order`（微信支付单号、状态、工期类型）。
- `event`：埋点流水（见 §六）。

**扣点时序（绝不能失败还扣钱，沿用塔罗落地包）**
`校验余额 → 预扣/冻结 → 算盘+调AI+选品 → 全部成功才确认扣点 / 任一失败立即返还`。
- 超时(建议20s)自动重试1次；AI JSON 解析失败重出1次→仍失败降级（展示已算的盘+相位+「解读没接上，积分已返还」）。
- 幂等：每次请求带 requestId，防重试重复扣点。

**新人**：注册送够 1 次合盘的积分（先尝「被说中」的甜头）。首串可设「合盘用户专享价/首单减」把人迈过付费门槛。

**定制品退换**：定制成串默认不支持无理由退（下单页明示）；质量问题按质保。现货/定制在商品与购物车分开标。

## 六、埋点 & P0 裁决看板

**必埋事件**：`funnel_enter, synastry_start, synastry_done, reco_view, preview_enter, add_cart, pay_success, join_private, repurchase`（均带 userId, ts, 关联id）。

**看板指标（对齐你的 P0 指标表）**

| 指标 | 算法 | P0 关注 |
|---|---|---|
| 各步转化率 | 下一步/上一步 | 盯 S3→S4→S6→S7 |
| 解读→加购率 | add_cart / synastry_done | 推荐引擎有没有说服力 |
| 加购→支付率 | pay_success / add_cart | 价格/工期/信任卡不卡 |
| 客单价 | 支付总额 / 付费订单 | 对串是否抬客单 |
| 复购率 | 二次付费人 / 首次付费人 | 命门，30%+ 强信号 |
| 私域转化率 | join_private / 用户 | 护城河，目标20%+ |
| **LTV/CAC** | LTV(≈客单×复购次数) / CAC | **>1 才继续放大** |

**两条红线**：LTV/CAC 持续 <1 → 停下改模型别加预算；复购长期个位数 → 先补私域/召回再放量。

## 七、技术要点 / 接口

- **合盘计算**：后端 Swiss Ephemeris 算行星黄经/宫位/相位（数据不出境，国内合规友好）。相位按容许度(orb)判定，权重见落地包。
- **AI 解读**：System = 落地包合盘 Prompt（含红线/分寸/范例，输出含 energy 的 JSON）；温度 0.7–0.85；只输出 JSON。
- **简化八字/五行**：出生信息 → 四柱 → 五行计数 → 最缺/最忌（P0 简化；接口 `getWuxing(profile)` 预留，P1 换精确 agent）。
- **选品引擎**：`selectBeads(vector, 净手围, 预算)` → 两串 skuSeq + reason（规则见 §四）。
- **前端**：小程序端 3D 星盘用小程序 WebGL canvas（复用 `synastry_reader` 场景）；成串预览用照片合成（复用 `bracelet_studio`，珠子资产换实拍/预渲染 PNG）。
- **支付**：微信支付；回调到账后写 ledger、置 order 状态；对账用四类流水。

## 八、验收标准（Definition of Done）

P0 上线 = 以下全绿：
1. 输入两个人（含 TA 不知时辰降级）能出**真盘**（换生日盘变）+ 跨盘相位 + 契合分 + 结构化解读（合规红线内）。
2. 自动生成「你的串/TA 的串」，推荐理由与解读一致，可预览/微调、选净手围/包装/证书。
3. 加购 → 微信支付 → 成功；扣点时序无「失败仍扣钱」；定制/现货工期正确标注。
4. 私域双入口 + 复购召回可用。
5. 九个埋点事件全部上报，裁决看板能算出各步转化率 + 客单价 + LTV/CAC。
6. 全程合规文案就位（§九）。

**Go/No-Go**：跑 2–4 周，看板 LTV/CAC 出结论。>1 放大；1–3 优化转化再放大；<1 停下改模型。

## 九、合规红线（写进产品）

- 合盘不预言关系结局、不替人做分合决定、不把相位读成宿命（困难相位=功课）。
- 香珠功用一律表述为**传统香方灵感 / 古法气味体验**，不碰「治/疗/改善睡眠」等医疗声称（国内广告法 + 为出海留一致口径）。
- 命理/选品全程「仅供自我觉察与决策参考」。
- 出生信息为敏感个人信息：明示用途、可删除；优先本地/境内计算。

## 十、给开发的交付方式

- 人类开发：本 PRD + 《合盘落地包》+ `synastry_reader.html` + `bracelet_studio` 一起给，沟通成本砍半。
- AI 编程（Claude Code/Cursor）：本 PRD 当规格，两个 html 当参考实现，落成小程序/H5。
- 建议节奏：P0-a 跑通合盘+解读+契合分（验「愿不愿为合盘付费」）→ P0-b 接选品+成串+支付（验「命理推荐→下单」）→ 私域+召回+看板。
