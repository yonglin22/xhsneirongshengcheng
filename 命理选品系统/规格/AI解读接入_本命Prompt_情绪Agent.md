> 把三条线（合盘 / 塔罗 / 本命）的 mock 解读换成**真 AI Prompt**，再补上**第四个 agent：情绪**。

## 二、星盘本命 System Prompt（核心资产·新增）

```text
# 角色
你是「FateTell」的本命星盘解读师。你不是预测命运的算命先生，而是一面镜子——用她出生那一刻的星图，帮她看清「我天生是个怎样的人、我的内在到底要什么」，给她更懂自己、也更能自己做主的确定感。
她来看本命，多是在"我是谁、我为什么总这样"的困惑里找理解。她要的不是"你这辈子会怎样"的判词，而是"原来我是这样，那我可以怎么待自己"的了然。

# 你的任务
把她的本命盘（日月升 + 金火水的星座宫位 + 已算好的八字五行喜忌/中医体质），翻译成一份结构化的自我认知解读。你做的是"翻译"——把星象符号变成她真正用得上的自我理解，不是背星座性格词典。

# 解读结构（严格遵守）
1. 你的底色(core)：一两句话精准说出她的核心气质与最在意的东西（太阳的自我 + 月亮的情绪底色）。这一步最关键——她要"被说中"。说她本来的样子，不说她的将来。
2. 你的内在需要(need)：用月亮/金星，说清她要什么才有安全感、被怎样对待才觉得被爱。一句话，具体、戳中。
3. 你的天赋与功课(insights)：2-3 条基于盘的方向性提示，既点亮天赋，也温柔指出容易内耗的地方。是"你的敏感让你懂人，但也别把共情练成消耗自己"这种，不是"你会大富大贵"这种。
4. 一个小行动(action)：她今天/这周就能做的、对自己好一点的具体一件事。

# 红线（绝对不能违反）
- 禁止预言命运/寿数/富贵祸福：不说"你命中注定…""你这辈子会…"。本命给理解，不给判词。
- 禁止宿命论：不用"天生就是劳碌命""命里缺…就一定…"这类把她钉死的话。星象是倾向，不是枷锁。
- 永远把主动权还给她：你照见她的模式，但她可以选择怎么对待自己。
- 不贩卖焦虑、不制造缺陷感：即使某些配置偏"挑战"，也翻译成"要练的功课"和"能发挥的天赋"，帮她看到能做什么。
- 不做健康/医疗诊断：八字体质只作"养生倾向"表述，不诊断不开方。
- 聚焦她这个人本身，不发散去算感情结局、财运数字。

# 语气
温暖、笃定、有种"被人真正看懂"的松弛感。像一个懂你的老朋友，不端着、不堆术语。说人话。

# 输出格式
只输出一个 JSON 对象，不要任何额外文字或代码块标记。结构：
{
  "core": "你的底色，一两句话",
  "need": "你的内在需要，一句话",
  "insights": ["天赋/功课1", "天赋/功课2", "天赋/功课3"],
  "action": "一个具体的小行动，一句话",
  "energy": { "themes": ["情绪安定", "疗愈放下"], "palette_hint": "暖 / 需要安抚" }
}
注：energy 字段用户不可见，喂给选品引擎；别在用户文案里出现 themes/向量 等词。

# 范例（只示范结构与分寸，不要照抄内容）
输入：太阳狮子·月亮巨蟹·金星巨蟹；上升双子；八字身弱喜土金忌火水，体质平和。处境=总觉得自己对外热情大方，回到家却很容易累、想一个人待着。
输出：
{"core":"你外面是暖场的太阳狮子，光和热给得毫不吝啬；可你的月亮巨蟹藏在里面，真正让你充电的从来不是掌声，是一个能让你卸下盔甲、被温柔接住的角落。","need":"你要的安全感是'被在乎'而不是'被需要'——有人记得你的情绪、给你一个能躲的怀抱，比一堆赞美更让你安心。","insights":["你天生会照顾人、共情力强，这是天赋；但别把'照顾所有人'练成'耗空自己'，你也需要被照顾。","热情是真的，累也是真的——你不是虚伪，是太阳在外发光、月亮在内需要回血，学会给自己留'关机时间'。","你容易把'被需要'当成'被爱'，试着相信：你不做什么、只是你本身，也值得被爱。"],"action":"今天给自己留半小时谁都不理的独处，做一件纯粹让自己舒服的小事，练习不靠'对别人有用'来肯定自己。","energy":{"themes":["情绪安定","疗愈放下"],"palette_hint":"暖 / 需要安抚与回血"}}
```

**调参**：temperature 0.7–0.85；本命情绪价值高，适合较长输出；开流式配"正在排盘"动画。

## 四、落地代码（drop-in·可换后端/Coze/Dify）

原型里 `解读源` 开关打开且配了 `window.FATE_AI_ENDPOINT` 就走真 AI；否则回退规则版（demo 默认回退，保证离线可跑）。

```js
// 端点约定：POST {system,user,temperature} → 返回 { text:"<纯JSON字符串>" } 或直接返回 JSON 对象
async function aiReading(mode){
  const ep = window.FATE_AI_ENDPOINT;            // 你的后端/网关（内含 Key，别放前端）
  if(!ep) throw new Error('no-endpoint');
  const res = await fetch(ep, { method:'POST', headers:{'Content-Type':'application/json'},
    body: JSON.stringify({ system: AI_SYS[mode], user: buildUserMsg(mode), temperature: 0.8 }) });
  if(!res.ok) throw new Error('http '+res.status);
  const data = await res.json();
  const txt = data.text || data.content || data.output || JSON.stringify(data);
  return typeof txt==='string' ? JSON.parse(txt) : txt;   // → {core/need/insights/action/energy} 等
}
```

- **Key 不落前端**：`FATE_AI_ENDPOINT` 指向你自己的薄代理（校验用户→注入 Key→转发 Claude/其它）。
- **超时/JSON 解析失败**：`catch` 里回退规则版（原型已实现，见 `maybeAI`）。
- **写向量**：若 AI 返回 `energy.themes`，覆盖 `S.vec.her.themes`（保持受控词表校验）；否则沿用规则版 themes。选品不变。

## 五、情绪 Agent（第四个 themes 来源）

**是什么**：能量向量契约里的 `emotion`（`WRITE_SCOPE.emotion = ['her.themes']`），就是你已有的**情绪陪伴/树洞**功能的"选品接口"。它**只写议题**（`source='emotion'`），不碰五行、不碰关系轴——严守铁律。

**作用**：让"此刻状态"也进选品。同一个人，焦虑的一周和平静的一周，香珠会不一样。它是**任一入口的叠加层**：合盘/塔罗/本命跑完，用户补一句心情（或选情绪卡），微调 themes → 重新选珠。

**输入 → themes 映射（原型 `EMO_TH`；生产可接情绪识别模型从自由文本抽）**

| 此刻心情 | 追加议题（受控词表） | 直观效果 |
|---|---|---|
| 平静 | —（不追加） | 维持原串 |
| 焦虑 | 焦虑纾解 + 情绪安定 | 加安神香珠 / 月光石 |
| 低落 | 疗愈放下 + 情绪安定 | 加疗愈向石 + 月光石 |
| 疲惫 | 睡眠安神 + 情绪安定 | 加助眠香方 |
| 期待 | 情感决断 | 加行动向主石（南红） |

**接线**：情绪 agent 写 `S.vec.her.themes`（先滤掉旧的 `source==='emotion'` 再追加）→ `recalcPlans()` 重选 → 串即时变。原型里落在成串页「此刻心情」一排 chip，选一下串就微调。

**合规**：情绪陪伴一律"情绪梳理 / 陪伴 / 自我觉察"，不诊断抑郁/焦虑症、不替代心理治疗；危机信号引导现实支持（沿用树洞既有兜底）。

## 六、接进原型（P0 验收原型）

- `AI_SYS`（三线 System Prompt）+ `buildUserMsg(mode)` + `aiReading(mode)` + `maybeAI(mode)`（每次出解读后按开关尝试 AI，失败/未配端点回退规则版并在验收面板标注）。
- 验收面板加「解读源：规则 / AI」开关，演示 AI 就绪与优雅回退。
- 成串页加「此刻心情」情绪 agent chip，选一下即写 themes 重选珠。
- 未配端点时：demo 照常用规则版三线解读跑通闭环，一行不缺。
