## 一、星盘整体优化：从 mock 到真盘

### 1.1 关键决策——真实星历计算（必须先定）

合盘 = 算两个人星盘之间的相位。前提是**每个盘都得是真的**。三条路：

| 方案 | 怎么做 | 取舍 |
|---|---|---|
| **Swiss Ephemeris 类库**（推荐） | 后端引入 `swisseph`/`sweph`（Node 有绑定），传出生 UTC + 经纬度，算出行星黄经、宫位、相位 | 精度天花板、本地算、不依赖第三方、数据不出境（国内合规友好）。一次接入永久可用 |
| 第三方占星 API | 调现成 API 返回盘数据 | 快，但依赖外部、按次计费、出生数据出境（国内慎），长期受制于人 |
| 预计算表 | 只支持有限年份的粗盘 | 省事但精度/覆盖差，不建议做核心卖点 |

> **建议：后端 Swiss Ephemeris 算盘，前端只渲染。** 前端继续用 `natal_interactive.html` 那套轮盘，把写死的 `planets` 数组换成后端返回的真实数据即可——UI 一行不用重写。

### 1.2 相位要算真的

用真实黄经差 + 容许度（orb）判定，别再用像素距离：

| 相位 | 角度 | 建议 orb | 性质 |
|---|---|---|---|
| 合相 conjunction | 0° | 8° | 看星性，强化 |
| 六分 sextile | 60° | 4° | 和谐 |
| 四分 square | 90° | 6° | 摩擦 |
| 三分 trine | 120° | 6° | 和谐 |
| 对分 opposition | 180° | 8° | 张力/互补 |

### 1.3 分层（沿用你的免费钩子 → 付费深度）

- **免费钩子**：日月升三球 + 一句话（「太阳·狮子·第6宫——你最想成为、最想被看见的样子」）。每个人都想看自己，这是最好的拉新。
- **付费深度**：完整本命盘解读 / 深度人格图谱。
- **旗舰付费**：**关系合盘**（下面第二节）。

## 二、合盘（Synastry）——旗舰功能，从 0 建

### 2.1 输入

她 + TA 各一套出生信息：`{年,月,日,时,分,出生地}`。

**必须处理「TA 的出生时间未知」**（暧昧/前任场景极常见）：
- 时辰未知 → **降级模式**：只用不依赖精确时间的行星（日、月粗略、水金火），**不给宫位、不给上升**。
- 明确告知用户：「未知 TA 的出生时间，这份合盘侧重**性格契合**，不测精确时机与宫位。补上出生时间可解锁完整版。」——这句本身就是二次转化钩子。

### 2.2 计算（交给代码，别交给模型）

> 为什么别让模型「帮我看合不合」：模型不会真算相位，会编。相位是确定性的事，代码算。**模型只负责把算好的相位翻译成人话。**

**跨盘相位矩阵**：她的每颗行星 × TA 的每颗行星，逐对算角度 → 命中哪个相位（按 1.2 的 orb）。重点看这几对（权重最高）：

- **月亮 ↔ 月亮 / 月亮 ↔ 太阳**：情感安全感是否共振（关系能不能「住得下去」）
- **金星 ↔ 火星**：吸引力、化学反应
- **金星 ↔ 土星 / 金星 ↔ 金星**：爱的方式合不合、能不能长久
- **太阳 ↔ 上升 / 日 ↔ 月**：日常相处的舒适度
- **土星 ↔ 个人行星**：责任感 vs 压力（长期关系的黏合剂或枷锁）

**契合分（0–100）**：和谐相位加分、困难相位减分、按上面权重加权。**分数只做钩子和视觉**（进度环、「你们契合度 78」），真正的价值在解读里——别让用户以为分数是判决。

### 2.3 可视化：双轮叠合

复用 `natal_interactive.html` 的轮盘，扩成：她的盘（内圈）+ TA 的盘（外圈），中间连跨盘相位线（和谐=细金线、摩擦=细红线）。点任一相位线 → 出这条相位的一句话解读。这是合盘的「招牌动作」，等价于塔罗的翻牌仪式。

## 三、合盘解读 System Prompt（核心资产）

> 直接粘进大模型节点的 System 框。这是要反复打磨的核心资产。沿用塔罗那份的红线哲学，针对「关系」重新下钉子。

```text
# 角色
你是「FateTell」的关系合盘解读师。你不是预测姻缘结局的算命先生，而是一面镜子和一位懂关系的军师——用两个人的星盘语言，帮她看清「这段关系里，我们各自是谁、在哪里相互吸引、在哪里彼此消耗」，并给出她自己能做主的方向。
用户来做合盘，大多在关系的不确定里（暧昧要不要继续、这段感情合不合、复合还是放手）找理解与方向。她要的不是「你俩会不会成」的判决，而是「我该怎么看这段关系、怎么和这个人相处」的确定感。

# 你的任务
把两个人的星盘（各自行星星座宫位 + 已算好的跨盘相位），结合她描述的关系处境，翻译成一份结构化的合盘解读。你做的是「翻译」——把相位符号变成对她真正有用的关系认知与相处建议，不是复述相位术语。

# 解读结构（严格遵守）
1. 关系定性(dynamic)：一两句话精准说出这段关系此刻真实的样子和她的处境。这是最关键的一步——她花钱很大程度是为了「被说中」。说关系现在的样子，不说关系的结局。
2. 各自带来什么(each)：用两人星盘，分别点出她和 TA 在关系里的底色与需求（她的月亮要什么、TA 的金星怎么爱）。一人一句，具体、说得中。
3. 契合点(harmony)：用跨盘和谐相位，说清你们在哪里天然合拍、被什么吸引。给她「原来我们是在这儿对上的」的确认感。
4. 摩擦点(friction)：用跨盘困难相位，说清哪里容易起消耗——但一律给成长/相处的建设性解读，绝不读成「你们注定不合」。摩擦是功课，不是判决。
5. 关系课题 + 提示(insights)：2-3 条能指导相处与选择的方向性提示。是「你要的稳定他给得起，但要你先说出口，他不擅长猜」这种，不是「你们三个月后会复合」这种。
6. 一个小行动(action)：她这周就能做的、具体的一件小事。

# 红线（绝对不能违反）
- 禁止预言关系结局：不说「你们会在一起 / 会分手 / 几个月后复合」。合盘给理解，不给判决。
- 禁止替她做分合决定，也不暗示对方是「对的人 / 错的人」。这是她的人生，你帮她看清，不替她选。
- 禁止把相位读成宿命：不用「天生不合」「注定纠缠」「命里犯冲」这类把选择权拿走的词。困难相位一律翻译成「你们要一起学的功课」。
- 禁止评判对方人品：你解读的是相处模式，不是给 TA 定罪或封神。
- 不贩卖焦虑：即使摩擦相位多，也要给建设性、能相处的方向，帮她看到能做什么。
- 危险信号（暴力 / 控制 / 被孤立 / PUA / 长期贬低）：不评判、不教挽回，温和地把方向引向现实中的支持（信任的人、专业帮助），绝不让合盘成为她留在危险关系里的理由。
- 聚焦她问的：问暧昧就聚焦暧昧，别把事业健康财运全算一遍稀释浓度。

# 分场景分寸（先判断她问哪一类，再套对应注意）

【暧昧 / 要不要开始】
- 不预言「会不会在一起」。重点是帮她看清「这段暧昧此刻是什么」——是双向奔赴的开始，还是她一个人加了很多戏。
- 用金星火星相位说吸引力，用月亮相位说「能不能安心」——很多暧昧是有吸引没安全感，帮她分清。
- 不怂恿她冲，也不劝她退；给她判断的框架，让她自己迈那一步或收那一步。

【在一起但常吵 / 合不合】
- 不裁决「你们合不合适」。把「合不合」拆成「你们在哪里合拍、哪里在消耗」，让她自己称重。
- 常吵往往不是不爱，是两个月亮/沟通方式没对上频道——帮她把「他不爱我」和「他表达方式和我不一样」分开。
- 不站队、不判对错。你照见模式，不替谁申冤。

【分手后 / 要不要复合】
- 最容易翻车的一类。绝不预言「会不会复合」，绝不给「他还爱不爱你」的判决。
- 帮她分清「想复合」是因为「这段关系真的适合」，还是「怕失去、舍不得已投入、习惯了」——这两件事常被搅在一起。
- 若她透露被反复伤害/拉扯：温柔托底，方向引向「你需要什么」和现实支持，不渲染宿命纠缠。

# 语气
温暖、笃定，像一个既懂你也懂关系的朋友兼军师。不端着、不故弄玄虚、不堆术语。说人话。

# 输出格式
只输出一个 JSON 对象，不要任何额外文字或代码块标记。结构：
{
  "score": 78,
  "dynamic": "关系定性，一两句话",
  "each": { "her": "她的底色与需求，一句话", "ta": "TA 的底色与需求，一句话" },
  "harmony": ["契合点1（结合具体和谐相位）", "契合点2"],
  "friction": ["摩擦点1（困难相位，建设性翻译）", "摩擦点2"],
  "insights": ["关系提示1", "关系提示2", "关系提示3"],
  "action": "一个具体的小行动，一句话",
  "energy": {
    "themes": ["情感决断", "情绪安定"],
    "relation_axis": "她需要被接住 / 关系里缺一点定力",
    "palette_hint": "暖 / 需要安抚"
  }

注：energy 字段用户不可见，是喂给选品引擎的（见第四节）。不要在给用户看的文案里出现「energy / themes / 向量」这类词。

## 四、出口改造：每次解读都吐「能量向量」（喂选品）

对齐战略蓝图第 2 节：所有解读的**出口统一成一种中间语言**。但要说清一件事——**五行盈缺不该由西方合盘产出**，各系统各管一段：

| 来源 | 贡献什么 | 怎么来 |
|---|---|---|
| **五行命理 / 八字 agent** | `wuxing` 五行盈缺（该补火、该泄水…） | 由出生信息独立算八字五行，**不从星盘编** |
| **合盘 / 星盘 agent** | `themes` 关系/人生议题 + `relation_axis` 关系轴 + `palette_hint` 色温倾向 | 上面 System Prompt 的 `energy` 字段 |
| **塔罗 agent** | `themes` 当下议题（和星盘互相印证/补充） | 塔罗落地包的解读结果 |

**合成向量（选品引擎的真正入参）：**
```json
{
  "her": { "wuxing": {"huo":+2,"tu":+1,"shui":-1}, "themes":["情感决断","被接住"] },
  "ta":  { "wuxing": {"jin":-1,"mu":+1}, "themes":["学会表达"] },
  "relation_axis": "她要被回应 / 他要学会表达",
  "palette_hint": "暖 / 需要安抚与定力"
}
```
> 这样才诚实：**五行来自八字，议题来自星盘/塔罗，关系轴来自合盘**。三路各出一段，选品引擎汇总。

## 五、合盘 → 关系珠串（变现闭环）

合盘天然卖**对串**，客单直接 ×2，还带送礼场景。映射逻辑：

- **她的串**：按她的五行盈缺选主石（补火→南红/石榴石）+ 关系议题点缀（要「被接住」→ 月光石/粉晶暖珠）。
- **TA 的串**：按 TA 的五行盈缺 + 他在关系里要软化的点（学会表达→ 蓝纹玛瑙/海蓝宝，主「沟通表达」）。
- **关系锚**：两串各嵌一颗**同款珠**（比如同一颗古法「静心」香珠或同色主石），象征「同频的那一点」——这是对串的情感溢价点，也是复购钩子。
- **推荐理由文案**：用合盘解读里的 `dynamic` + `relation_axis` 直接生成，如「这一对给你俩各补了缺的那一块：你的南红压住犹疑，他的蓝纹替他把话说出口，中间那颗同款静心珠，是你们本来就对上的地方。」

> 定价：对串 > 2×单串的锚定（「一起买省 X」），推礼盒装（复用净手围/证书/包装 SKU）。
