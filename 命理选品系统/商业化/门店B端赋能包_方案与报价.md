# 门店 B 端赋能包 · 方案与报价（国内 B2B）

> 定位：**你是军火商，不是零售商。** 把「命理选品系统 + 供应链 + 内容 + 分销」打包成门店 SaaS，门店做脸、你收赋能费与货差。玄学的流量与合规风险落在门店端，不直接压在你头上。
> 表内价格均为**起始假设（示例，待用真实成本/竞品校准）**，不是定价定案；结构先定死，数字后调。

---

## 一、门店拿到什么（赋能包 = 四件套 + 一套 SOP）

| 模块 | 门店得到 | 你的壁垒 |
|---|---|---|
| ① 白牌小程序 | 门店自有品牌的命理选品小程序：**关系合盘 / 本命星盘 / 塔罗**（解读引擎）+ **选品成串**（向量→珠串）+ 商城 + 微信支付 + 会员/积分 | 别的选品系统只有拼串，没有命理内核 |
| ② 供应链 | **香承古法香珠 + 水晶**按拿货价供货，可门店备货或**一件代发**；证书/包装配套 | 香承真实货源（已做代理/跨境），逐 SKU 打好五行/议题标签 |
| ③ 内容操盘台 | 开放本平台「赛道智能体 · 小红书内容操盘台」：给门店源源产小红书/本地种草图文 | 别人给不了的获客赋能 |
| ④ 代理分销 | 驻场设计师入驻 + 多级代理分销（抄 Stone LAB 机制），门店可发展下线 | B 端裂变自带 |
| ⑤ 培训/SOP | 玄学解读话术、成串引导话术、**合规红线 SOP**、选品逻辑、私域运营手册 | 把"能开单"标准化，降低门店门槛 |

**一句话卖点（对门店）**：不用懂命理、不用自己拍内容、不用压货，一套小程序 + 一条供应链 + 一台内容机，把「进店/加微的女性客户」变成「按她命理排的珠串订单」。

## 二、分层套餐

| 套餐 | 面向 | 白牌小程序 | 供货政策 | 内容额度/月 | 分销权限 | SaaS 费(示例/年) |
|---|---|---|---|---|---|---|
| **体验版** | 单店试水 | 标准模板、门店 logo | 一件代发（无需备货） | 20 篇 | 无 | ¥ 3,000 |
| **标准版** | 单门店 | 白牌 + 自定义皮肤 | 代发 + 拿货价备货 | 60 篇 | 一级分销 | ¥ 9,800 |
| **旗舰版** | 连锁/多门店 | 多门店隔离、总部后台 | 更低拿货价 + 优先货期 | 200 篇 | 多级分销 | ¥ 29,800 |
| **区域代理** | 渠道商 | 可再开白牌给下游门店 | 区域独家拿货价 | 定制 | 全级 + 招商分成 | 面议 + 保证金 |

> 阶梯逻辑：低门槛先让门店跑起来（体验版几乎零风险，代发不压货），跑通再升级压货/分销拿更低货价——把门店从"试用"推向"深度绑定"。

## 三、你的收入结构（四条线）

1. **SaaS 订阅费**：按套餐年费（见上），旗舰/代理更高。稳定现金流。
2. **供应链差价**：门店售价 − 门店拿货价。核心利润池（香珠/水晶差价 + 证书/包装加项）。
3. **分销抽成**：门店/代理产生的分销 GMV 抽成（示例 5–10%）。
4. **增值服务**：内容超额包、精品皮肤、1 对 1 人工解读分成（林老师那类高客单，平台抽成）。

> 口径：SaaS 保底、货差走量、分销放大。**别把利润全压在 SaaS**——门店最认的是"能不能帮我卖货"，SaaS 是入场券，货差与分销才是规模利润。

## 四、门店经济模型（让门店看到赚头 · 示例待校准）

单个标准版门店，一个月：

| 项 | 示例值 | 说明 |
|---|---|---|
| 进店/加微女性客 | 300 人 | 门店自有客流 + 内容拉新 |
| 合盘/选品体验率 | 40% → 120 人 | 免费钩子（合盘/日签）拉动 |
| 下单转化率 | 12% → 36 单 | 命理推荐→对串下单 |
| 客单价 | ¥ 220 | 对串抬客单（她+TA） |
| **门店月销售额** | **≈ ¥ 7,920** | 36 × 220 |
| 门店拿货成本 | ≈ ¥ 3,200 | 约 40% 货成本 |
| **门店月毛利** | **≈ ¥ 4,720** | 销售额 − 货成本 |
| 分摊 SaaS | ¥ 817 | 9,800/12 |
| **门店月净赚** | **≈ ¥ 3,900** | 一个店一个月回本还有得赚 |

**你从这一个店赚**：SaaS 817/月 + 货差（拿货 3,200 里你的加价部分）+ 分销抽成。**关键裁决**：单店模型能否标准化复制 → 能，才谈规模（对齐战略蓝图 P2）。

## 五、落地 SOP（门店 onboarding，7 天开张）

```
D0 签约/选套餐 → D1 开通白牌(门店号/logo/皮肤) → D2 铺 SKU(选主推香珠/水晶+标签) →
D3 培训(解读话术+成串引导+合规红线+私域) → D4 内容操盘台产首批种草 →
D5 试运营(店员实操合盘→出串→下单) → D7 正式开张 + 首月扶持(内容/话术答疑)
```

首月扶持是留存关键：帮门店出前几单，让店主看到"真能卖"，续费与升级自然来。

## 六、代理分销机制（抄 Stone LAB，B 端裂变）

- **层级**：平台 → 区域代理 → 门店 → 门店的分销员/KOC。
- **抽成**：逐级设佣（示例：门店分销员拿 GMV 10%，区域代理拿下游 GMV 3–5% 招商分成）。
- **结算**：小程序内记录归因（谁的码/链接），T+N 结算，走微信支付分账。
- **驻场设计师**：优质晒单/客订进"灵感池"，带货分成——UGC 即销售。

## 七、白牌技术要点（一套系统多门店）

- **多租户**：单套系统多门店隔离——门店号(tenantId) 贯穿数据/订单/结算；皮肤/品牌可配置。
- **数据隔离**：各门店客户/订单/私域资产互不可见；总部(连锁)可看下辖门店汇总。
- **供货/分账**：拿货价按套餐分级；分销归因 + 微信支付分账；对账用四类流水（充值/消费/赠送/返还）。
- **内容操盘台对接**：门店在后台一键调赛道智能体产内容（复用本 repo 的 S1–S8 流水线 + 知识库）。
- **合盘/选品**：复用《合盘落地包》+《能量向量数据契约》+《P0 PRD》；后端 Swiss Ephemeris 算盘，选品引擎读向量出串。

## 八、合规与风控（B 端责任划分）

- **玄学合规**：门店做脸→经营主体是门店，但你**必须**给合规 SOP（不预言/不宿命/不替人做分合决定、财运不碰标的）+ 话术红线，写进培训与小程序文案，降低整体投诉/封号风险。
- **香珠功用**：一律"传统香方灵感/古法气味体验"，不碰医疗声称（广告法）。
- **供应链**：香承手工定制**周期约 15–20 天、产能有限**——现货/定制分开标；水晶配鉴定证书；定制成串退换规则前置写死。
- **数据**：出生信息为敏感个人信息，明示用途可删除，境内计算。

## 九、招商节奏 & 裁决

- **先签 3–5 家试点门店**（不同城市/客群），跑 1–2 个月，验单店模型。
- **裁决指标**：单店月净赚是否 > 门店预期 & 你的**单店贡献（SaaS+货差+分销）/ 获客该门店的成本 > 1**；单店 SOP 能否标准化复制。
- 达标 → 上区域代理放大；不达标 → 先改单店模型（选品转化/客单/内容获客），别急着招商铺量。
