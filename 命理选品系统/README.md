## 怎么用（三层关系）

1. **先读战略蓝图** —— 明确定位、四智能体如何合成一次推荐、双市场打法、Stone LAB 抄什么/跳过什么、路线图与合规红线。
2. **再读规格** —— 三份是「单一可信源」，照着建即可：
   - 塔罗 / 合盘两份落地包 = **解读引擎**（红线哲学一致：只翻译不预言，把选择权还给用户）。
   - P0 PRD = **第一条能赚钱的漏斗**（合盘 → 关系对串），含选品引擎「向量→SKU」映射规则、扣点闭环、埋点与 **LTV/CAC** 裁决口径。
3. **参考实现直接拿去改** —— 两个 HTML 自包含、可直接用浏览器打开：
   - `关系合盘_3D参考实现.html`：换生日盘会变（真算，非 mock），可拖动旋转星盘、点亮行星；开发者面板可看喂选品的向量 JSON。
   - `成串照片合成台.html`：珠子为预渲染示意资产，**换成实拍图即照片级**；内含拍摄资产规范。

## 架构命门：能量向量中间层（已定死，见 规格/ + 契约/）

所有智能体把结果吐进**同一种中间语言**：`五行·能量向量 + 议题标签 + 关系轴`。
- 五行盈缺 ← 八字/五行命理（五行唯一真值来源）
- 议题 ← 塔罗 + 星盘/合盘（受控词表，禁自由造词）
- 关系轴 ← 合盘

选品引擎只干一件事：把向量映射到打了同样标签的 SKU（石头/香珠）。做好这层，才是「一串为你而排的珠」，而不是四个各玩各的玄学玩具。
**铁律**：agent 之间不越界（塔罗不写五行）；补五行绝不落在忌神上；缺八字则降级为纯议题选石。
契约与代码：`规格/能量向量中间层_数据契约.md`、`契约/energy_vector.ts`、`契约/energy_vector.schema.json`。
