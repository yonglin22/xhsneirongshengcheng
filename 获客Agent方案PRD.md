# 获客 Agent 产品方案 / PRD

> 版本 v0.1（2026-06）｜负责人：孙永林
> 定位：朱砂·操盘台「一站式 AI 流量分发平台」的第二引擎。**内容 Agent 负责生产，获客 Agent 负责分发与养号获客。**

---

## 1. 背景与目标

### 1.1 背景
内容 Agent（yonglin.chat）已上线，能 3 分钟产出可发布的小红书笔记 / 公众号文章。但"生产出来"只是一半——商家真正的痛点是：
- **不会养号**：新号没权重，一发就限流；
- **没法规模化**：要做矩阵（多账号 / 多平台），靠人工一个个发，累且乱；
- **分发割裂**：小红书、公众号各发各的，没有统一台子。

### 1.2 目标
把"内容生产 → 多账号矩阵分发 → 养号获客"接成闭环，让一个人也能运营一个号矩阵。

| 维度 | 目标 |
|---|---|
| 用户价值 | 一处生产，一键分发到多账号草稿箱；自动养号降低限流/封号 |
| 业务目标 | 提升内容 Agent 的留存与付费（分发是高频刚需，带动订阅）|
| 2026 目标 | 把"小红书养号 + 矩阵分发（到草稿箱）+ 公众号分发"做到可用、稳定 |

### 1.3 北极星与关键指标
- 北极星：**周活跃分发账号数**（接入并成功分发的账号数 × 周分发次数）
- 辅助：养号任务完成率、分发成功率、号存活率（未限流/封禁占比）、内容→分发转化率

---

## 2. 用户与场景

| 用户 | 场景 | 诉求 |
|---|---|---|
| 中小商家 / 个体 | 自己做 3~10 个小红书号 + 1 个公众号 | 内容批量铺、号别养死 |
| 内容创业者 / 起号者 | 测多个赛道/人设 | 矩阵起号、低成本试错 |
| 代运营 / 工作室 | 管几十个客户号 | 集中管理、批量分发、看状态 |

核心场景串联：
> 在内容 Agent 出一篇 → 选择要分发的账号（矩阵）→ 一键推送到各账号草稿箱 → 各号人工终审后发布 → 养号 Agent 后台维持账号活跃与权重。

---

## 3. 产品定位与边界

### 3.1 与内容 Agent 的关系
```
内容 Agent（生产）          获客 Agent（分发/养号）
选题→…→正文→封面    ──►   账号矩阵 · 一键分发到草稿箱 · 养号任务
                          └─►  公众号草稿箱 / 小红书草稿箱
```

### 3.2 明确边界（合规与风控红线）
- **只分发到「草稿箱」，不做全自动发布**：终审与"发布"动作由用户在各平台 App 完成（降低风控与合规风险，也符合平台规则）。
- **养号 = 辅助拟真行为 + 节奏建议**，不承诺"绝对不封号"；高风险动作（批量关注、刷量）默认不做。
- 不碰：买卖账号、刷粉刷量、违规引流话术。

## 4. 功能模块

### 4.1 模块一：账号矩阵管理（地基）
所有功能的前提是"把号接进来、管起来"。

- **账号接入**
  - 小红书：复用现有方案——用户在本机/受控环境登录 → 同步登录态（cookie/token）到服务端账号库（已有 xhs-cli + cookie 同步链路）。
  - 公众号：走**官方授权**（公众号 appid + secret / 第三方平台授权），拿到调用草稿箱接口的权限。
- **账号档案**：平台、昵称、头像、赛道/人设标签、绑定的内容 Agent 赛道、健康状态、登录态有效期。
- **分组**：按赛道 / 客户 / 平台分组，批量操作。
- **健康监控**：登录态是否失效、近期是否疑似限流（看互动异常）、需重新登录提醒。

### 4.2 模块二：小红书养号 Agent
目标：让账号"像真人在用"，提升权重、降低新号/沉睡号被限流概率。

- **养号任务（拟真行为，受控、低频、可关）**
  - 浏览/停留信息流、按关键词搜索浏览、点赞/收藏少量同赛道优质笔记、关注少量同行（每日上限严格、随机化节奏）。
  - 完善资料引导（头像/简介/标签符合人设）。
- **养号节奏方案**：按"新号期 / 成长期 / 成熟期"给不同强度的日任务模板；用户可一键应用 + 手动调。
- **风控护栏**：所有动作有每日上限、随机间隔、模拟真人时段；异常（验证码/风控提示）即暂停并通知用户。
- **养号日报**：今日完成了哪些动作、账号状态、建议。

> ⚠️ 技术现实：小红书无官方开放接口，养号靠自动化操作真实登录态，**有风控风险**。策略是"低频拟真 + 用户可控 + 出风控即停"，把风险降到可接受，而非声称零风险。

### 4.3 模块三：内容矩阵分发（核心高频）
把内容 Agent 的成稿，批量推送到所选账号的**草稿箱**。

- **分发入口**：内容 Agent 成稿页新增「分发到矩阵」；或获客 Agent 内选历史成稿。
- **选号**：勾选目标账号（可按分组全选），跨平台混选。
- **按平台适配**：
  - 小红书：标题/正文/图片/标签 → 推到该号草稿箱。
  - 公众号：标题/正文（含 ## 小标题排版）/首图/摘要 → 推到该号草稿箱（官方草稿接口）。
- **一稿多发的差异化（可选进阶）**：同一篇给不同号做轻度改写（换钩子/换开头），降低平台判重、贴合不同人设。
- **分发记录**：每次分发的账号、状态（成功/失败/待重试）、草稿链接、时间，失败可重试。

### 4.4 模块四：公众号分发（独立可先落地）
公众号有**官方草稿箱 API**，是三块里技术最确定、合规最稳的，建议优先落地。
- 新增草稿（draft/add）：标题、作者、正文 HTML、封面素材（先上传永久素材拿 media_id）、摘要、原文链接。
- 草稿管理：列表、更新、删除。
- 用户在公众号后台/小程序终审后群发。

## 5. 核心流程

```
① 接入账号（小红书同步登录态 / 公众号授权）
        │
②（可选）开启养号 Agent → 后台按节奏跑拟真任务 + 日报
        │
③ 内容 Agent 出稿
        │
④「分发到矩阵」→ 选号（跨平台）→ 按平台适配 → 推到各草稿箱
        │
⑤ 用户在各平台 App 终审 + 发布（人工把关，合规）
        │
⑥ 分发记录 / 养号日报 → 复盘
```

## 6. 技术方案要点与风险

| 模块 | 方案 | 风险/约束 | 对策 |
|---|---|---|---|
| 小红书接入 | 登录态（cookie/token）同步到服务端账号库（已有链路） | 登录态会过期；数据中心 IP 登录易触发验证码 | 居民环境登录后同步；过期提醒重登；可配代理 |
| 小红书养号/分发 | 自动化操作真实账号（web/cli） | **无官方 API、风控风险** | 只到草稿箱、低频拟真、出风控即停、严格上限 |
| 公众号分发 | 官方草稿箱 API（draft + 素材） | 需公众号授权；正文需转公众号 HTML；图片需先传素材 | 官方授权流程；正文 Markdown→公众号 HTML 转换器；封面自动传永久素材 |
| 多账号并发 | 任务队列 + 每账号串行限速 | 并发触发风控 | 队列限速、随机抖动、分时段 |
| 安全 | 登录态/密钥加密存储 | 账号被盗风险 | 加密存储、最小权限、审计日志 |

> 关键判断：**公众号分发最先做（官方 API 稳）**；小红书"分发到草稿箱 + 轻养号"次之（风险可控）；全自动发布/重度养号不做。

## 7. 数据模型（简）

- **account**：id, user_id, platform(xhs/gzh), nickname, avatar, track_tag, persona, auth_blob(加密登录态/授权), auth_expire_at, health_status, created_at
- **account_group**：id, user_id, name, account_ids[]
- **nurture_task**：id, account_id, type(浏览/点赞/关注/资料), plan_stage, daily_quota, status, last_run_at, log
- **distribute_job**：id, user_id, content_id, account_id, platform, payload(适配后的标题/正文/图), status(pending/success/fail), draft_url, error, created_at
- **content**：复用内容 Agent 的成稿（topic/title/body/cover/summary/platform）

## 8. MVP 范围与里程碑（2026 路线）

### M1 · 地基 + 公众号分发（最先落地，约 4~6 周）
- 账号矩阵管理（接入/分组/健康监控）
- 公众号授权 + 一键分发到草稿箱（官方 API）
- 内容 Agent 成稿页「分发到矩阵」入口
- 验收：能把一篇公众号文章一键推到 N 个公众号草稿箱

### M2 · 小红书分发到草稿箱（约 4~6 周）
- 小红书账号接入（登录态同步）+ 批量推草稿箱
- 分发记录 / 失败重试
- 验收：一篇笔记一键推到多个小红书号草稿箱

### M3 · 小红书养号 Agent（约 4~6 周）
- 养号任务（低频拟真）+ 节奏模板 + 风控护栏 + 日报
- 验收：账号可按节奏自动维持活跃，出风控即停并通知

### M4 · 进阶
- 一稿多发差异化改写、定时分发、矩阵数据看板（各号互动/涨粉）

---

## 9. 合规与风险声明
- 仅分发到草稿箱，发布由用户人工完成；不代发、不刷量、不买卖账号。
- 养号为拟真辅助，**不承诺不封号**；产品内明确风险提示与用户授权。
- 登录态/授权信息加密存储，用户可随时解绑。
- 内容侧沿用内容 Agent 的合规自检（违禁/导流/绝对化）。

---

## 10. 衡量指标
- 接入账号数、周活跃分发账号数、分发成功率、草稿→发布转化（可让用户回填）
- 养号任务完成率、账号存活率（未限流/封禁）
- 对内容 Agent 的拉动：开通分发用户的留存/付费 vs 未开通

## 11. 待决策问题
1. 小红书"推草稿箱"的技术可达性需先做**可行性验证**（当前自动化能否稳定写入草稿箱？还是只能落地到"内容包导出 + 半自动"）。
2. 公众号授权用**自有服务号/第三方平台**哪种接入成本低、可规模化？
3. 养号强度默认值、各平台每日动作上限的安全阈值（需小范围实测）。
4. 矩阵分发的计费模型（按账号数/按分发次数/订阅档）。
