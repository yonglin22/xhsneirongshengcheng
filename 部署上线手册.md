# 朱砂 · 操盘台 — 部署上线手册

> 面向 1:1 正式上线。本项目 = 零依赖 Node HTTP 服务（`server.js`，静态托管 + AI 代理）+ `node:sqlite`（`billing.db`）。

---

## 0. 环境要求
- **Node ≥ 22**（用到 `node:sqlite`，本机已用 v24，OK）。
- 一台 Linux 服务器（2C2G 起）、一个**已 ICP 备案**的域名、SSL 证书。
- 各 AI 服务的 key（DeepSeek / 智谱 GLM-4V / Seedream(ark) / 可灵 Kling / Exa）已充值。

---

## 1. 上线必改的环境变量（`.env`）
```bash
NODE_ENV=production                      # 关闭联调后门（dev-confirm/devPay 会 403）
BILLING_ENABLED=true                     # 开计费
SESSION_SECRET=<64位强随机串>            # 改掉！否则会话可伪造/重启掉登录
COOKIE_SECURE=true                       # HTTPS 下会话 Cookie 带 Secure
PUBLIC_BASE_URL=https://你的域名          # 海报/落地页二维码用它，朋友才能扫开
ADMIN_TOKEN=<强随机串>                    # 别用 admin888
ADMIN_PHONES=你的管理员手机号

# 短信（二选一，必须真发，否则注册登录走不通）
SMS_PROVIDER=aliyun
SMS_WEBHOOK_URL=https://你的短信中转/api/send   # 方式①：收到 {phone,code,sign,template} 后调厂商
SMS_SIGN=朱砂操盘台
SMS_TEMPLATE=SMS_xxxxxx
# 方式②：直接在 server.js 的 sendSms() 里接阿里云/腾讯云 SDK

# 支付（一期：扫码+客服开通）
PAY_WECHAT_QR=/assets/pay-qr.png         # 换成真实收款码
SUPPORT_WECHAT_QR=/assets/support-qr.png # 真实客服码
SUPPORT_WECHAT_ID=你的客服微信号
```
生成强随机串：`node -e "console.log(require('crypto').randomBytes(48).toString('base64url'))"`

> ⚠️ `.env` 权限 `chmod 600 .env`，不要进 Git。

## 2. 短信验证码（登录命脉）· 腾讯云
代码已内置腾讯云 `SendSms`（TC3-HMAC-SHA256 签名，零依赖）+ **频率限制**（同号 60s、同 IP 每小时 10 次，返回 429）。设了 `SMS_PROVIDER=tencent` 后验证码**只发不回显**；发送失败接口返回 502（不假装成功）。

**`.env` 配置：**
```bash
SMS_PROVIDER=tencent
TENCENT_SECRET_ID=         # 访问管理 → API 密钥
TENCENT_SECRET_KEY=
TENCENT_SMS_SDK_APPID=1400xxxxxx   # 短信→应用管理 SDKAppID
TENCENT_SMS_SIGN=朱砂操盘台         # 已审核通过的签名内容
TENCENT_SMS_TEMPLATE=2000xxx       # 已审核通过的正文模板ID
TENCENT_SMS_MINUTES=5              # 模板含「{2}分钟有效」才填；只有{1}则留空
TENCENT_SMS_REGION=ap-guangzhou    # 可选，默认广州
```

**腾讯云控制台开通步骤（链接见下）：**
1. **创建签名**（签名管理）：类型选「网站/APP/公众号/小程序」，提交资质，审核约 1-2 小时。
2. **创建正文模板**（正文模板），推荐文案：`您的验证码为{1}，{2}分钟内有效，请勿泄露。`（→ 配 `TENCENT_SMS_MINUTES=5`）。
3. **创建短信应用**拿 `SDKAppID`（应用管理）。
4. **建 API 密钥** SecretId/SecretKey（访问管理 CAM）。
5. 填 `.env` → `pm2 restart zhusha` → 真机自测。

> 控制台链接：
> - 签名管理：https://console.cloud.tencent.com/smsv2/csms-sign
> - 正文模板：https://console.cloud.tencent.com/smsv2/csms-template
> - 应用管理(SDKAppID)：https://console.cloud.tencent.com/smsv2/app-manage
> - API 密钥(CAM)：https://console.cloud.tencent.com/cam/capi

**自建中转（备选）**：设 `SMS_WEBHOOK_URL`，你的服务收到 `POST {phone,code,sign,template}` 后自行调任意厂商发送。

## 4. Nginx 反向代理 + HTTPS
```nginx
server {
  listen 443 ssl http2;
  server_name 你的域名;
  ssl_certificate     /etc/ssl/你的.crt;
  ssl_certificate_key /etc/ssl/你的.key;

  client_max_body_size 30m;          # 允许图片上传
  gzip on; gzip_types text/css application/javascript application/json image/svg+xml;

  location / {
    proxy_pass http://127.0.0.1:8787;
    proxy_set_header Host $host;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;   # 供限流取真实IP
    proxy_set_header X-Forwarded-Proto $scheme;
    proxy_read_timeout 120s;          # AI 生成较慢，给足超时
  }
}
server { listen 80; server_name 你的域名; return 301 https://$host$request_uri; }
```
> 静态资源已**全部本地化**（Tailwind / qrcode / html2canvas / jszip 都在 `assets/`，无外网 CDN，国内可访问）。

## 5. 数据库与备份（billing.db = 钱包/订单/作品唯一真相源）
- 放在持久盘；**每日备份**（含 `-wal`/`-shm`）：
```bash
# crontab -e
0 4 * * * cd /path/to/art-grad-frontend && cp billing.db "/backup/billing-$(date +\%F).db" 2>>/var/log/zhusha-bak.log
```
- 迁移/还原：停服 → 拷 `billing.db*` 三个文件 → 启服。

## 8. 待决策 / 后续（不阻断上线，但建议尽快）
- **生成图过期**：Seedream/可灵返回的是签名临时 URL，会过期 → 老作品的真·AI 配图日后失效；长期保存需落 OSS/本地存储（长图笔记是 CSS 渲染，不受影响）。
- **xhs 实时检索**：依赖服务器上 `xhs` CLI 的小红书登录态，会失效需维护；不维护就只留 Exa 网页检索。
- **多实例**：当前限流/会话用进程内存，单机够用；要多实例需迁 Redis + 共享 DB。
- **法务**：用户协议、隐私政策、支付/发票资质。
