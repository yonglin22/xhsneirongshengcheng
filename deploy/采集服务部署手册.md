# 小红书采集服务（collector）VPS 部署手册

目标：在一台常开 VPS 上跑 `collector.js`（封装已登录的 `xhs` CLI），对外开 HTTPS 接口；
Render 主站用 `COLLECTOR_URL` 调它，实现**正式环境按维度抓真实笔记**（最热/最新/综合 × 图文/视频 × 翻页）。

> 选 **香港 VPS**（腾讯云轻量·香港 / 阿里云香港等，约 ¥24/月）：免备案、访问小红书稳定。
> 域名用 `collector.yonglin.chat`（在你的 DNS 加一条 A 记录指向 VPS 公网 IP）。

---

## 0. 准备（腾讯云轻量·香港）
- 买**腾讯云轻量应用服务器**，地域选**香港**，镜像选 **Ubuntu Server 22.04 LTS**（纯系统，不要选带面板的应用镜像），最低配（1核1G/2G）即可。
- ⚠️ **关键：腾讯云轻量的防火墙在控制台**，不是只靠系统 ufw。进 控制台 → 你的轻量实例 → **防火墙** → 添加规则，放行：
  - `TCP 22`（SSH）、`TCP 80`（证书申请用）、`TCP 443`（HTTPS）。
  - 不放行 80/443 → Caddy 申请证书会失败、Render 也连不上。
- 用控制台的"重置密码"设 root 密码，或用密钥登录 SSH。
- 设一个**强随机密钥**当 `COLLECTOR_TOKEN`（`openssl rand -hex 24` 生成），记下来，VPS 和 Render 两边要一致。
- DNS：在你管理 yonglin.chat 的地方加一条 A 记录 `collector` → VPS 公网 IP（香港 VPS 免备案）。

## 1. 装依赖（Node + pipx + xhs CLI + Caddy）
```bash
# Node 20+
curl -fsSL https://deb.nodesource.com/setup_20.x | bash -
apt-get install -y nodejs python3 python3-venv pipx git
# Caddy（自动 HTTPS）
apt-get install -y debian-keyring debian-archive-keyring apt-transport-https curl
curl -1sLf 'https://dl.cloudsmith.io/public/caddy/stable/gpg.key' | gpg --dearmor -o /usr/share/keyrings/caddy-stable-archive-keyring.gpg
curl -1sLf 'https://dl.cloudsmith.io/public/caddy/stable/debian.deb.txt' | tee /etc/apt/sources.list.d/caddy-stable.list
apt-get update && apt-get install -y caddy
```

## 2. 建专用用户 + 拉代码
```bash
useradd -m -s /bin/bash collector
mkdir -p /opt/xhs-collector
git clone https://github.com/yonglin22/xhsneirongshengcheng.git /tmp/repo
cp /tmp/repo/collector.js /opt/xhs-collector/
chown -R collector:collector /opt/xhs-collector
```

## 3. 装 xhs CLI 并扫码登录（关键：用扫码，不碰 cookie）
```bash
su - collector
pipx install xiaohongshu-cli
pipx ensurepath && source ~/.bashrc
xhs login --qrcode          # 终端会打印二维码，用【手机小红书 App → 我 → 扫一扫】扫它
xhs status                  # 显示已登录用户信息即成功
exit
```
> 登录态会过期（一般数天~两周）。过期后重跑 `su - collector` → `xhs login --qrcode` 扫一次即可，主站无需改动。
> 采集服务的 `/health` 接口能看 `loggedIn` 状态。

## 4. 配 systemd 常驻
```bash
cp /tmp/repo/deploy/collector.service /etc/systemd/system/collector.service
# 编辑把 COLLECTOR_TOKEN 改成你的强随机密钥：
nano /etc/systemd/system/collector.service
systemctl daemon-reload
systemctl enable --now collector
systemctl status collector          # active (running) 即可
curl -s http://localhost:8799/health   # 看 loggedIn:true
```

## 5. 配 HTTPS 反代（Caddy）
先在 DNS 加 A 记录：`collector.yonglin.chat → VPS 公网 IP`（等生效）。
```bash
cp /tmp/repo/deploy/Caddyfile.collector /etc/caddy/Caddyfile
# 确认里面域名是 collector.yonglin.chat
systemctl reload caddy
# 验证（外网）：
curl -s https://collector.yonglin.chat/health
```

## 6. Render 配置（让线上主站用它）
Render → 你的服务 → Environment 加两个：
```
COLLECTOR_URL   = https://collector.yonglin.chat
COLLECTOR_TOKEN = 第0步那个强随机密钥（和 VPS 上一致）
```
保存→自动重部署。之后线上"抓对标"就走 VPS 采集服务，抓真实笔记。

## 7. 验证全链路
线上点"抓对标"，或直接：
```bash
curl -s -X POST https://你的Render域名/api/xhs-search \
  -H 'content-type: application/json' \
  -d '{"keyword":"美术考研","sort":"popular","type":"image","page":1}'
```
返回 `ok:true` + 20 篇真实笔记即成功。

## 排查
| 现象 | 原因/处理 |
|---|---|
| `/health` loggedIn:false | 没登录/已过期 → `su - collector` 后 `xhs login --qrcode` 重扫 |
| 主站报"采集服务不可达" | DNS 没生效 / Caddy 未起 / 防火墙没放行 80,443 |
| 主站报"采集服务 401" | 两边 `COLLECTOR_TOKEN` 不一致 |
| 抓取偶发为空/验证码 | 抓太频繁；已有间隔。香港 IP 比海外稳；必要时降频 |
| 升级 collector.js | VPS 上 `git -C /tmp/repo pull && cp /tmp/repo/collector.js /opt/xhs-collector/ && systemctl restart collector` |
